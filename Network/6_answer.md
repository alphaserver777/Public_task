# Вопрос 6: Назначение консольного кабеля. Режимы доступа Cisco IOS и их возможности. Running-config и startup-config. Автодополнение команд. Команда `no`

---

#### Подробный ответ:

**1. Консольный кабель**

* **Назначение:** подключение к коммутатору или маршрутизатору напрямую для конфигурации устройства.
* Используется **только для управления устройством**, не для передачи данных в сети.
* Чаще всего **RS-232 (RJ-45 ↔ DB9) или USB** на современных устройствах.
* Подключение к консоли позволяет получить доступ к **CLI (Command Line Interface)**.

---

**2. Режимы доступа Cisco IOS**

Cisco IOS имеет **несколько режимов работы**, каждый с разными правами:

1. **Пользовательский режим (User EXEC Mode)**

* Префикс: `Router>`
* Возможности: базовая диагностика, просмотр конфигурации, проверка состояния интерфейсов.
* Ограничен для конфигурации.

2. **Привилегированный режим (Privileged EXEC Mode)**

* Префикс: `Router#`
* Доступ ко всем командам, включая сохранение и загрузку конфигурации.
* Команды: `show running-config`, `reload`, `copy running-config startup-config`.

3. **Режим глобальной конфигурации (Global Configuration Mode)**

* Префикс: `Router(config)#`
* Настройка устройства в целом: интерфейсы, VLAN, протоколы.
* Вход: из привилегированного режима `configure terminal`.

4. **Режим специфической конфигурации (Interface / Line Configuration Mode)**

* Префикс: `Router(config-if)#`
* Настройка конкретного интерфейса или линии консоли/VTY.
* Пример: настройка IP на интерфейсе или пароля на VTY.

---

**3. Running-config и startup-config**

* **Running-config** — текущая конфигурация в оперативной памяти (RAM).

* Сбрасывается при перезагрузке.
* **Startup-config** — сохранённая конфигурация на NVRAM.

* Загружается при включении устройства.
* Для сохранения изменений:

```text
Router# copy running-config startup-config
```

---

**4. Автодополнение команд**

* В Cisco IOS при вводе команд можно использовать **Tab**, чтобы автодополнить или подсказать команды.
* Позволяет быстрее вводить команды и уменьшить ошибки.

---

**5. Команда `no`**

* Используется для **отмены настройки или возврата к значению по умолчанию**.
* Примеры:
```text
Router(config)# no shutdown   # активировать интерфейс (если он был выключен)
Router(config)# no ip route 192.168.1.0 255.255.255.0 10.0.0.1
Router(config)# no vlan 10
```
**Краткий ответ для Anki:**

* **Консольный кабель** — прямое управление устройством.
* **Режимы IOS:**

* User EXEC (`>`), Privileged EXEC (`#`), Global Config (`(config)#`), Interface Config (`(config-if)#`).
* **Running-config** — текущая конфигурация; **startup-config** — сохранённая.
* **Tab** → автодополнение.
* **`no`** → отмена команды / возврат к умолчанию.
