# Вопрос 8: Алгоритм разбиения адресного пространства IPv4 на подсети

---

#### Подробный ответ:

Разбиение (subnetting) позволяет **разделить сеть на несколько подсетей**, чтобы оптимально использовать адреса и повысить управляемость сети.

---

**1. Шаг 1: Определяем исходные параметры**

* **Сеть**: исходный адрес сети, например `192.168.1.0/24`.
* **Количество подсетей**: сколько подсетей нужно.
* **Количество хостов на подсеть**: минимальное количество устройств, которое должно помещаться.

---

**2. Шаг 2: Определяем, сколько бит нужно для подсетей**

* Используется формула:
[
    2^n \geq \text{кол-во подсетей}
]
где **n** — количество бит, добавляемых к части сети.

**Пример:**

* Нужно 4 подсети → (2^n \ge 4 \Rightarrow n = 2) бита.

---

**3. Шаг 3: Определяем, сколько бит нужно для хостов**

* Для хостов:
[
    2^h - 2 \ge \text{кол-во хостов на подсеть}
]
где **h** — количество бит, оставляемых под хосты.
* Вычитаем 2 адреса: **адрес сети** + **broadcast**.

**Пример:**

* Нужны 50 хостов → (2^h - 2 \ge 50 \Rightarrow h = 6) (т.к. (2^6-2=62)).

---

**4. Шаг 4: Определяем новую маску подсети**

* Маска = исходная маска + bиты для подсети.
* Пример: исходная `/24`, добавляем 2 бита для подсети → новая маска `/26`.

---

**5. Шаг 5: Определяем адреса подсетей**

* Адрес каждой подсети = базовый адрес сети + шаг (increment) по количеству хостов.
* Шаг = (2^h) (число адресов в подсети).

**Пример:** `192.168.1.0/24` → /26 (64 адреса на подсеть):

    | Подсеть | Адрес сети    | Broadcast     | Доступные хосты               |
    | ------- | ------------- | ------------- | ----------------------------- |
    | 1       | 192.168.1.0   | 192.168.1.63  | 192.168.1.1 – 192.168.1.62    |
    | 2       | 192.168.1.64  | 192.168.1.127 | 192.168.1.65 – 192.168.1.126  |
    | 3       | 192.168.1.128 | 192.168.1.191 | 192.168.1.129 – 192.168.1.190 |
    | 4       | 192.168.1.192 | 192.168.1.255 | 192.168.1.193 – 192.168.1.254 |

    ---

    **6. Шаг 6: Проверка**

    * Каждая подсеть имеет:

    * уникальный адрес сети
    * уникальный broadcast
    * нужное количество хостов
    * Подсети не пересекаются.

    ---

    **Краткий ответ для Anki:**

    1. Определить исходную сеть и требуемые подсети.
    2. Вычислить биты для подсетей: (2^n \ge \text{кол-во подсетей}).
    3. Вычислить биты для хостов: (2^h-2 \ge \text{хостов}).
    4. Новая маска = исходная + биты подсети.
    5. Адреса подсетей = базовый + шаг (2^h).
    6. Проверка: уникальный адрес сети, broadcast, нужные хосты, подсети не пересекаются.
