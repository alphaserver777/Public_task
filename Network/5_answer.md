# Вопрос 5: Принцип работы коммутатора. Определение и назначение VLAN. Access и trunk-порты

---

#### Подробный ответ:

**1. Принцип работы коммутатора (Switch)**

* Коммутатор работает на **канальном уровне OSI** (Data Link Layer).
* Основная задача: **пересылка Ethernet-кадров на правильный порт по MAC-адресу**.
* Механизм работы:

1. **Обучение MAC-адресов:**

* Коммутатор записывает в таблицу (MAC → порт), с какого порта пришёл кадр.
2. **Пересылка кадров:**

* Если MAC получателя известен → пересылает только на нужный порт (unicast).
* Если MAC неизвестен → пересылает на все порты кроме входного (flooding).
* Broadcast и multicast пересылаются на все соответствующие порты.
* **Цель:** уменьшить коллизии, повысить пропускную способность сети, обеспечить сегментацию трафика.

---

**2. VLAN (Virtual Local Area Network)**

* **VLAN** — виртуальная локальная сеть внутри физического коммутатора.
* **Назначение:**

* Сегментация сети без физического разделения.
* Повышение безопасности и управления трафиком.
* Разделение трафика разных отделов или сервисов.
* Каждому VLAN присваивается **номер (ID)**, например VLAN 10, VLAN 20.

**Принципы работы VLAN:**

* Устройства одного VLAN видят друг друга как в одной сети.
* Трафик разных VLAN не пересекается без маршрутизатора или L3-коммутатора.
* Маркировка кадров VLAN осуществляется тегом **802.1Q**.

---

**3. Порты коммутатора: Access и Trunk**

1. **Access-порт:**

* Подключение **одного устройства или одной VLAN**.
* Кадры без тегов VLAN → коммутатор добавляет VLAN ID внутрь.
* Пример: ПК сотрудника, IP-телефон.
* Команда Cisco:

```text
Switch(config)# interface FastEthernet0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10
```

2. **Trunk-порт:**

* Подключение **другого коммутатора или маршрутизатора**, передающего **несколько VLAN** одновременно.
* Использует **802.1Q тегирование**, чтобы различать VLAN.
* Пример: соединение двух коммутаторов между этажами.
* Команда Cisco:

```text
Switch(config)# interface GigabitEthernet0/1
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk allowed vlan 10,20,30
```

---

**Краткий ответ для Anki:**

* **Коммутатор:** пересылает кадры по MAC, уменьшает коллизии.
* **VLAN:** виртуальная сеть → сегментация и безопасность.
* **Access-порт:** один VLAN → ПК, телефон.
* **Trunk-порт:** несколько VLAN → соединение коммутаторов, теги 802.1Q.
