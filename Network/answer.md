# Вопрос 1: Уровни модели OSI и их назначение. Уровни модели TCP/IP. Назначение между уровнями OSI и TCP/IP.

#### Подробный ответ:

**1. Модель OSI (Open Systems Interconnection)**

Модель OSI — это эталонная модель, описывающая, как данные передаются в компьютерных сетях. Она разделена на **7 уровней**, каждый из которых выполняет определённые функции:

1. **Физический уровень (Physical layer)**

* Отвечает за передачу необработанных битов по физической среде.
* Определяет электрические, оптические, радиосигналы, разъёмы, кабели.
* Примеры: витая пара, оптоволокно, коаксиальный кабель, Wi-Fi.

2. **Канальный уровень (Data Link layer)**

* Организация передачи данных между узлами в пределах одной сети.
* Обеспечивает обнаружение и коррекцию ошибок.
* Управляет MAC-адресами.
* Примеры: Ethernet, PPP, VLAN, коммутаторы.

3. **Сетевой уровень (Network layer)**

* Определяет маршруты и адресацию пакетов между сетями.
* Управляет IP-адресами, маршрутизацией.
* Примеры: IPv4, IPv6, маршрутизаторы, протоколы ICMP, OSPF.

4. **Транспортный уровень (Transport layer)**

* Обеспечивает надежную передачу данных между приложениями.
* Разделяет данные на сегменты, управляет порядком доставки и проверкой ошибок.
* Примеры: TCP, UDP.

5. **Сеансовый уровень (Session layer)**

* Управляет сессиями (соединениями) между приложениями.
* Поддерживает открытие, управление и закрытие сеансов связи.
* Примеры: RPC, NetBIOS.

6. **Представительский уровень (Presentation layer)**

* Отвечает за кодирование, шифрование и преобразование данных.
* Обеспечивает совместимость форматов между системами.
* Примеры: ASCII, JPEG, MPEG, SSL/TLS.

7. **Прикладной уровень (Application layer)**

* Обеспечивает доступ приложений к сети.
* Примеры: HTTP, FTP, SMTP, DNS, Telnet.

**Назначение между уровнями:**
Каждый уровень OSI предоставляет сервисы уровню выше и использует сервисы уровня ниже. Это создаёт чёткую иерархию: физический уровень передаёт биты → канальный формирует кадры → сетевой маршрутизирует пакеты → транспорт гарантирует доставку сегментов → сеансовый управляет соединениями → представительский форматирует данные → прикладной предоставляет сервис пользователю.

---

**2. Модель TCP/IP (Интернет-модель)**

Модель TCP/IP более практична и используется в современных сетях. Она имеет **4 уровня**:

1. **Сетевой интерфейс (Network Interface / Link layer)**

* Совмещает функции физического и канального уровней OSI.
* Примеры: Ethernet, Wi-Fi.

2. **Интернет-уровень (Internet layer)**

* Соответствует сетевому уровню OSI.
* Обеспечивает маршрутизацию пакетов между сетями.
* Примеры: IPv4, IPv6, ICMP.

3. **Транспортный уровень (Transport layer)**

* Соответствует транспортному уровню OSI.
* TCP — надёжный поток данных, UDP — быстрый, но без гарантий доставки.

4. **Прикладной уровень (Application layer)**

* Объединяет прикладной, сеансовый и представительский уровни OSI.
* Примеры: HTTP, FTP, DNS, SMTP.

---

**3. Соответствие между уровнями OSI и TCP/IP**

| OSI               | TCP/IP            |
| ----------------- | ----------------- |
| Прикладной        | Прикладной        |
| Представительский | Прикладной        |
| Сеансовый         | Прикладной        |
| Транспортный      | Транспортный      |
| Сетевой           | Интернет          |
| Канальный         | Сетевой интерфейс |
| Физический        | Сетевой интерфейс |

---

**Краткий ответ для Anki:**

* OSI: 7 уровней — Физический, Канальный, Сетевой, Транспортный, Сеансовый, Представительский, Прикладной.
* TCP/IP: 4 уровня — Сетевой интерфейс, Интернет, Транспорт, Прикладной.
* OSI ↔ TCP/IP: Физ+Канальный → Сетевой интерфейс, Сетевой → Интернет, Транспорт → Транспорт, Прикладной+Представительский+Сеансовый → Прикладной.
