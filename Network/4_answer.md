# Вопрос 4: MAC-адресация. Unicast, broadcast и multicast рассылка. Медиаконвертер. Коммутатор. Формат Ethernet-кадра. Обнаружение ошибок с помощью алгоритма CRC32.

---

#### Подробный ответ:

**1. MAC-адресация**

* **MAC-адрес (Media Access Control address)** — уникальный идентификатор сетевого интерфейса в локальной сети.
* Формат: 48 бит (6 байт), обычно пишется как `00:1A:2B:3C:4D:5E`.
* Назначение: адресация на канальном уровне, чтобы коммутатор понимал, куда отправлять кадры.

**Типы рассылки:**

1. **Unicast** — кадр отправляется на конкретный MAC-адрес (один получатель).
2. **Broadcast** — кадр отправляется всем узлам в сети (`FF:FF:FF:FF:FF:FF`).
3. **Multicast** — кадр отправляется группе устройств, подписанных на определённый адрес (`01:00:5E:xx:xx:xx` для IPv4).

---

**2. Медиаконвертер**

* Устройство для преобразования сигнала между разными средами передачи:

* Например, витая пара ↔ оптоволокно.
* Позволяет интегрировать разные типы сетей без замены оборудования.

---

**3. Коммутатор (Switch)**

* Работает на **канальном уровне OSI**.
* Функции:

* Читает MAC-адреса входящих кадров.
* Формирует таблицу MAC → порт.
* Пересылает кадры только на порт назначения (unicast) или всем портам (broadcast).
* Отличие от хаба: коммутатор уменьшает коллизии и увеличивает пропускную способность.

---

**4. Формат Ethernet-кадра**

| Поле                        | Размер       | Назначение                                  |
| --------------------------- | ------------ | ------------------------------------------- |
| Преамбула                   | 7 байт       | Синхронизация передачи                      |
| Start Frame Delimiter (SFD) | 1 байт       | Начало кадра                                |
| Destination MAC             | 6 байт       | MAC получателя                              |
| Source MAC                  | 6 байт       | MAC отправителя                             |
| EtherType / Length          | 2 байта      | Тип протокола (IPv4, IPv6) или длина данных |
| Data / Payload              | 46–1500 байт | Передаваемые данные                         |
| FCS (CRC32)                 | 4 байта      | Контрольная сумма для проверки ошибок       |

---

**5. Обнаружение ошибок: CRC32**

* **CRC32 (Cyclic Redundancy Check)** — циклический избыточный код на 32 бита.
* Принцип:

* Отправитель вычисляет CRC по данным и добавляет в FCS.
* Получатель пересчитывает CRC и сравнивает.
* Если CRC не совпадает — кадр считается повреждённым и отбрасывается.

---

**Краткий ответ для Anki:**

* **MAC-адрес** — уникальный идентификатор интерфейса.
* **Unicast** → один, **Broadcast** → всем, **Multicast** → группе.
* **Медиаконвертер** → преобразует среду передачи.
* **Коммутатор** → пересылает кадры по MAC, уменьшает коллизии.
* **Ethernet-кадр:** Преамбула, SFD, MAC-адреса, EtherType, Payload, FCS (CRC32).
* **CRC32** → проверка ошибок передачи.
